# Blog Cheatsheet

This post shows every MDX feature available. Flip `draft: true` to `false`
in the frontmatter to publish it, or keep it as a private reference by
visiting `/blog/cheatsheet` directly while previewing locally.

## Text basics

You can write **bold**, *italic*, ~strikethrough~, and `inline code`. Links
work out of the box: [Ntlaks.dev](https://ntlaks.dev) opens inline, and
external links are auto-configured with `target="_blank"`.

> Blockquotes pull important lines out of the flow. Use them sparingly for
> emphasis or to quote someone.

## Lists

Unordered:

- First point
- Second point
- Nested items work too
  - Sub-point A
  - Sub-point B

Ordered:

1. Set up the project
2. Ship a first version
3. Iterate with feedback

## Callouts

Four callout types exist. Pass a `type` prop.

<Callout type="info">
  Use **info** for general notes and context. The default type.
</Callout>

<Callout type="tip">
  Use **tip** for helpful advice, shortcuts, or "try this" moments.
</Callout>

<Callout type="warning">
  Use **warning** when something might bite: footguns, browser quirks,
  gotchas.
</Callout>

<Callout type="danger">
  Use **danger** for things that will definitely break: security issues,
  destructive actions.
</Callout>

You can also override the label with `title="..."`:

<Callout type="tip" title="Keyboard shortcut">
  Press <Kbd>Ctrl</Kbd> + <Kbd>K</Kbd> to quickly jump to the search bar.
</Callout>

## Code blocks

Plain fenced blocks get syntax highlighted:

```js
function sumOfSquares(n) {
  let total = 0;
  for (let i = 1; i <= n; i += 1) {
    total += i * i;
  }
  return total;
}
```

Many languages are supported (`js`, `ts`, `jsx`, `tsx`, `glsl`, `c`, `java`,
`python`, `sql`, `html`, `css`, `bash`):

```glsl
// vertex.glsl
attribute vec3 aPosition;
uniform mat4 uModelView;
uniform mat4 uProjection;

void main() {
  gl_Position = uProjection * uModelView * vec4(aPosition, 1.0);
}
```

Inline code like `const x = 1;` is styled in cyan to stand out.

## Images

Place image files in `public/images/blog/` and reference them with an
absolute path. Images auto-style with rounded borders and lazy loading.

### Plain image

```md
![Alt text describing the image](/images/blog/example.png)
```

### Figure with caption

Use the `Figure` component when you need a caption:

```mdx
<Figure
  src="/images/blog/shader-diagram.png"
  alt="Fragment shader pipeline diagram"
  caption="The fragment shader runs once per pixel after rasterization."
/>
```

## Video

### Self-hosted video (for short demos)

Drop an `.mp4` or `.webm` into `public/videos/` and embed it:

```mdx
<Video
  src="/videos/shader-demo.mp4"
  poster="/images/blog/shader-poster.jpg"
  caption="Live distortion shader reacting to pointer position."
/>
```

### YouTube embed

For longer videos, embed from your YouTube channel. Grab the video ID (the
string after `v=` in the URL) and pass it to the `YouTube` component:

```mdx
<YouTube id="dQw4w9WgXcQ" title="Optional accessible title" />
```

## Tables

Standard Markdown tables work and get styled by the prose plugin:

| Primitive         | Vertices needed | Typical use       |
| ----------------- | --------------- | ----------------- |
| `POINTS`          | 1 per point     | Scattered dots    |
| `LINES`           | 2 per line      | Disconnected lines|
| `TRIANGLES`       | 3 per triangle  | Solid shapes      |
| `TRIANGLE_STRIP`  | n + 2           | Connected surface |

## Mixing everything

Here is what a real teaching post might look like:

<Callout type="tip">
  Before you read, open the repo in a second tab:
  [ntlaks/WebGL-NDC-Visualizer](https://github.com/2Ntlaks/WebGL-NDC-Visualizer).
</Callout>

When you draw in WebGL, every coordinate you pass must live in a small cube
from `-1` to `+1` called the **Normalized Device Coordinate** (NDC) space.
Here is how you set a vertex at the top-right corner:

```js
const vertices = new Float32Array([
   1.0,  1.0, 0.0, // top-right corner in NDC
]);
```

## Publishing checklist

1. Set `draft: false` in the frontmatter
2. Run `npm run dev` and open `/blog/<slug>` to preview
3. Fix anything weird
4. `git add`, commit, push to `main`
5. Netlify rebuilds automatically
